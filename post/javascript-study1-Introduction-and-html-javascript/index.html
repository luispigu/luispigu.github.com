자바스크립트 제대로 배우기 1일차

이 포스팅은 페이스북 자바스크립트 제대로 배우기 그룹의 공부 내용을 요약한 것입니다.
출처 : <<프론트엔드 개발자를 위한 자바스크립트>>(2013 인사이트, 한선용 옮김)

1장 자바스크립트란 무엇인가?

자바스크립트의 간단한 역사
  • 1995년 자바스크립트 처음 등장했을 때 주요 목적은 입력 유효성 검사
  • 넷스케이프에서 일하던 브랜든 아이흐(Breadan Eich)가 Mocah, 나중엔 LiveScript라 불리는 스크립트 언어 개발
  • 경쟁사인 마이크로소프트에서 JScript 라는 이름의 자바스크립트를 포함한 IE3 출시
  • 자바스크립트 표준화 필요성 증가
  • 1997년 자바스크립트 1.1이 ECMA(유럽컴퓨터제작자협회)에 제안
  • JavaScript 1.1 을 손질해 ECMA-262 를 만들었고 이 표준은 ECMAScript(에크마스크립트)라는 이름의 스크립트 언어를 정의
자바스크립트 구현은 다음 세가지로 나뉨
  • 코어 (ECMAScript) : 단순히 명세에서 설명하는 부분들을 구현한 언어 (문법, 타입, 선언문, 키워드, 예약어, 연산자, 객체)
  • 문서 객체 모델 (DOM) : XML을 HTML에서 사용할 수 있도록 확장한 API
  • 브라우저 객체 모델 (BOM)
ECMAScript
  • ECMAScript 3.1 = ECMAScript-262 4판, 2009년 12월 3일 공식 발표
  • 2008부터 주요 웹 브라우저(인터넷 익스플로러, 파이어폭스, 사파리. 크롬, 오페라)는 모두 ECMA-262 3판을 준수
DOM(Document Object Model)
  • DOM API를 통해 노드를 쉽게 제거하고, 추가하고, 교체하고, 수정할 수 있다.
  • DOM level은 1부터 3까지 존재, 3으로 갈수록 확장 지원
BOM(Browser Object Model)
  • 브라우저 창에 접근하고 조작할 수 있게 하는 인터페이스
  • BOM을 이용하면 브라우저에 표시된 페이지와는 별개의 컨텍스트에서 브라우저와 상호작용할 수 있다.
  • HTML5에서 BOM을 공식 명세의 일부로 표준화 하고 있음.

2장 HTML 속의 자바스크립트

Script 요소

자바스크립트를 HTML 페이지에 삽입하는 일차적인 방법은 <script> 요소를 통하는 것

6가지 속성 존재
  • async - 옵션. 스크립트를 즉시 내려받지만 자원을 내려받거나 다른 스크립트를 불러오는 등 다른 페이지 작업을 방해해서는 안된다고 지시. 외부 스크립트 불러올 때만 유효
  • charset - 옵션. src 속성으로 명시한 코드의 문자셋을 지정. 브라우저는 대개 이 속성의 값을 무시하므로 좀처럼 쓰이지 않는다
  • defer - 옵션. 문서의 콘텐츠를 완전히 파싱하고 표시할 때까지 스크립트 실행을 지연해도 안전함을 나타냄. 외부 스크립트 파일을 불러올 때만 유효. IE7과 이전 버전에서는 인라인 스크립트에서도 이 속성을 허용
  • language - 폐기됨. 원래는 코드 블록에서 사용한 스크립트 언어를 "JavaScript", "JavaScript1.2" 처럼 나타낼 목적. 그러나 현재 대부분의 브라우저에서 이 속성을 무시하기에 써서는 안됨
  • src - 옵션. 실행할 코드를 포함한 외부 파일의 위치를 지정
  • type - 옵션. language 속성을 대체할 의도로 만들어짐. 스크립트 언어의 콘텐츠 타입(마임 타입)을 지정. 아직 습관적으로 또는 브라우저 호환성을 위해 "text/javascript"라 표기하곤 함. 사실은 생략해도 기본 값이므로 안전함
<script>요소는 두가지 방법으로 사용
  1. 페이지에 직접 작성
  2. 외부 파일에서 불러오기

스크립트 요소 내부의 자바스크립트 코드는 위에서부터 차례로 해석된다.
인라인 자바스크립트코드, 외부 파일의 코드 해석하는 동안에는 페이지 처리가 멈춤.

  • 페이지에 직접 작성한 예
<script type="text/javascript">
    function sayHi(){
    alert("Hi!");
</script>
  • 외부파일에서 불러온 예
<script type="text/javascript" src="example.js"></script>

스크립트 요소에서 가장 강력하면서도 논란이 끊이지 않는 부분은 다른 도메인에 있는 자바스크립트의 파일을 불러오는 능력(src 주소에 다른 도메인의 파일 입력)
주의 사항 : 다른 도메인에서 자바스크립트 파일을 가져올 때는 직접 관리하는 도메인이나 신뢰할 수 있는 곳에서 관리하는 도메인의 파일만 이용해야 한다.

태그 위치

전통적으로 페이지의 <head>요소 안에 쓰는 것이 일반적 : CSS파일이나 자바스크립트 파일 같은 외부 파일에 대한 참조를 한곳에서 관리하자는 목적
단점 : 자바스크립트 파일을 전부 <head>요소에서 불러오게 되는 경우 자바스크립트 코드를 전부 내려받고, 파싱하고, 해석을 끝낼 때까지 페이지 렌더링이 멈추게 된다.
위와 같은 이유로, 최신 웹 애플리케이션에서는 일반적으로 다음과 같이 자바스크립트 코드를 모두 <body>요소 안에, 페이지 콘텐츠 마지막에 쓴다.

스크립트 처리 지연

HTML 4.01 에서는 <script>요소에 defer라는 속성을 정의
defer의 목적은 스크립트에서 페이지 구조를 바꾸지 않는다고 명시하는 것
페이지 전체를 파싱한 후에 스크립트를 실행해도 상관 없음
defer 속성을 설정하면 브라우저는 해당 요소를 만나는 즉시 코드를 내려받지만 실행은 지연

두개의 외부 스크립트를 쓰는 경우 이론적으로는 스크립트 명세 순서대로 차례로 실행됨.
하지만 항상 순서대로 실행되지는 않으며 가능하다면 <script>요소는 하나만 쓰는 것이 최선. 이 내용은 HTML5 에서 명세화된 것이므로 HTML5를 지원하는 브라우저는 모두 인라인 스크립트의 defer 속성을 무시.
브라우저 마다 지원 여부가 다름. 따라서 지연시킬 스크립트는 페이지 맨 마지막에 놓는 것이 여전히 최상.

비동기 스크립트

HTML5에서 <script> 요소에 도입된 async 속성.
외부 스크립트에만 적용되며 defer속성과 비슷한 방법으로 스크립트를 처리
목적은 스크립트를 모두 내려받아 실행할 때까지 기다릴 필요 없이 페이지 렌더링을 시작해도 좋으며 앞선 스크립트 파일을 기다리지 않고 뒤에 있는 스크립트 파일을 내려받아 실행해도 좋다고 명시하는 것
주의 : DOM을 조작하는 스크립트는 비동기적async 로 불러오지 않는 편이 좋음

XHTML에서 자바스크립트 사용하기

CDATA와 자바스크립트 주석 기호를 같이 사용

<script type="text/javascript">
//<![CDATA[
function compare(a, b){
    if(a < b){
        alert("A is less than B");
    }else if(a > b){
        alert("A is greater than B");
    }else{
        alert("A is equal to B");
    }
}
//]]>
</script>

이렇게 작성하는 경우 최신 브라우저 전체에서 모두 잘 동작함.

구식 문법

스크립트 요소를 지원하지 않는 브라우저에서 스크립트 요소 내부 콘텐츠를 보이지 않게 하는 것.
다음과 같이 스크립트 코드를 HTML 주석으로 감싸는 방법

<script><!--
function sayHi(){
    alert("Hi!");
}
//--></script>

하지만 더 이상 필요하지 않으므로 써서는 안된다.

인라인 코드와 외부 파일

일반적으로 가능한 한 자바스크립트는 외부 파일로 분리하는 방법을 모범 사례로 간주.
외부 파일을 쓰면 다음과 같은 이점이 존재

  • 관리하기 쉽다 - 자바스크립트 파일을 모두 디렉터리 하나에 모아 놓으면 마크업과 무관하게 자바스크립트 코드를 수정가능하므로 훨씬 편리
  • 캐싱 - 서로 다른 페이지에서 같은 파일을 사용한다면 한 번만 내려받으면 되므로 페이지를 불러오는 시간이 짧아짐
  • 미래에도 안전하다 - CDATA나 주석 편법을 쓰지 않아도 된다
