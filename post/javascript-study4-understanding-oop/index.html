자바스크립트 제대로 배우기4 - 객체에 대한 이해

자바스크립트 제대로 배우기4 - 객체에 대한 이해

이 포스팅은 페이스북 자바스크립트 제대로 배우기 그룹의 공부 내용을 요약한 것입니다.
출처 : <<프론트엔드 개발자를 위한 자바스크립트>>(2013 인사이트, 한선용 옮김)

6.1 객체에 대한 이해

  • ECMAScript 에는 클래스 라는 개념이 없으며, EMCAScript 의 객체는 다른 클래스 기반 언어와 다름
  • 객체 : 프로퍼티의 순서 없는 컬렉션이며 각 프로퍼티는 원시 값이나 객체, 함수를 포함
객체 생성 :객체를 만드는 방법은 두가지가 있음
  1. Object의 인스턴스를 만들고 프로퍼티와 메서드를 추가
  2. 객체 리터럴 패턴 사용

현재는 아래와 같은 객체 리터럴 패턴이 많이 사용됨

var student = {
    name : Luis,
    age : 28,
    id : 12345678,
    sayName: function(){
        alert(this.name);
    }
};
프로퍼티 타입
  • 프로퍼티에는 데이터 프로퍼티, 접근자(accessor) 프로퍼티 두가지 타입이 존재
  • ECMAScript 5판 명세에 정의된 바에 따르면, 이들 속성은 자바스크립트에서 직접적으로 접근할 수 없음
데이터 프로퍼티
  • 데이터 값에 대한 단 하나의 위치를 포함하여 그 값을 읽고 씀

  • 4가지 속성 존재

    • [[Configurable]] : 해당 프로퍼티가 삭제, 변경, 변환 가능함을 의미. 기본값 true
    • [[Enumarable]] : for-in 루프에서 해당 프로퍼티를 반환. 나열 할 수 있음을 의미. 기본값 true
    • [[Writable]] : 프로퍼티의 값을 바꿀 수 있음. 기본값 true
    • [[Value]] : 프로퍼티의 실제 데이터 값. 기본값 undefined

  • 기본 프로퍼티의 속성을 바꾸려면 ECMAScript 5판의 Object.defineProperty(); 메서드를 사용

    • 프로퍼티를 추가하거나 수정할 객체, 프로퍼티 이름, 서술자(descriptor) 객체를 매개변수로 받음
    • 이들 값의 일부 또는 전부를 바꿔서 대응하는 속성 값 변경 가능
    • 예를 들어, writable : false 로 바꾸는 경우 읽기전용 프로퍼티를 의미
    • configurable을 false로 설정하면 해당 프로퍼티를 객체에서 제거할 수 없음
    • [[configurable]]을 false로 설정하면 다시는 수정할 수 없게 됨
    • Object.defineProperty() 호출 시 각 데이터 프로퍼티의 값을 따로 명시하지 않는다면 기본 값은 false
    • 주의 : IE8 에서는 Object.defineProperty()가 제대로 구현되지 않았으므로 사용 자제
접근자 프로퍼티
  • 접근자 프로퍼티는 값이 들어있지 않고 대신 두가지 함수가 포함 (getter, setter)
  • 4가지 속성 존재
    • [[Configurable]] : 해당 프로퍼티가 삭제, 변경, 변환 가능함을 의미. 기본값 true
    • [[Enumarable]] : for-in 루프에서 해당 프로퍼티를 반환. 나열 할 수 있음을 의미. 기본값 true
    • [[Get]] : 프로퍼티를 읽을 때 호출하는 함수. 기본값 undefined
    • [[Set]] : 프로퍼티를 바꿀 때 호출하는 함수. 기본값 undefined
  • 접근자 프로퍼티의 속성을 바꾸려면 Object.defineProperty()메서드를 사용
    • 접근자 프로퍼티는 일반적으로, 프로퍼티의 값을 바꿨을 때 해당 프로퍼티만 바뀌는 게 아니라 부수적인 절차가 필요한 경우 사용
    • _ 가 붙은 프로퍼티는 객체의 메서드를 통해서만 접근할 것이고 객체 외부에서는 접근하지 않겠다는 의도를 나타낼 때 쓰는 표기법
    • ECMAScript 5판 이전에는 두 가지 비표준 메서드 __defineGetter__()와 __defineSetter__() 를 이용해 접근자 프로퍼티를 생성
    • Object.defineProperty()를 지원하지 않는 브라우저에서 [[Configurable]]과 [[Enumarable]]을 수정할 수 있는 방법은 없음
다중 프로퍼티 정의
  • 여러 개의 프로퍼티를 동시에 수정해야 할 때는 Object.defineProperties() 메서드를 사용
  • 앞에서 사용한 Object.defineProperty()와의 차이점은 여러 프로퍼티를 동시에 생성했다는 것 뿐임
프로퍼티 속성 읽기
  • ECMAScript 5의 Object.getOwnPropertyDescriptor() 메서드를 이용해 원하는 프로퍼티의 서술자 프로퍼티를 읽을 수 있음
  • Object.getOwnPropertyDescriptor(읽을 프로퍼티가 있는 객체, 서술자를 가져올 프로퍼티) 형태로 사용
  • 반환값은 접근자 프로퍼티는 configurable, enumerable, get, set 을 프로퍼티로 포함하는 객체를 반환
  • 데이터 프로퍼티는 configurable, enumerable, writable, value 를 프로퍼티로 포함하는 객체를 반환
  • Object.getOwnPropertyDescriptor() 메서드는 자바스크립트 모든 객체에서 사용가능, DOM, BOM 객체도 해당됨
