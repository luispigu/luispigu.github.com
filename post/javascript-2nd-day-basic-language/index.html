자바스크립트 제대로 배우기 2일차

이 포스팅은 페이스북 자바스크립트 제대로 배우기 그룹의 공부 내용을 요약한 것입니다.

3장 언어의 기초

이 장을 읽기 전엔 주 내용인 문법이나 데이터 타입, 제어문 등이
어차피 이전에 다른 언어 공부한 경험이 있기 때문에 쉬울 거라 생각했다.
하지만 자바스크립트 만의 특징이랄까 정말 다른 부분이 많아서 생각보다 읽는데 너무 오래걸렸다.
그래서 기존 언어와 좀 다르고 많이 헷갈리는 부분 위주로 정리를 했다.

아래 내용은 ECMAScript 3판에서 정의된 내용을 바탕으로 함.

3.1 문법

  • 변수나 함수 이름, 연산자 모두 대소문자를 구분
  • ECMAScript 식별자는 관습적으로 카멜 케이스 사용
Strict 모드
  • 기존과는 다른 방식으로 좀 더 엄격하게 자바스크립트를 파싱하고 실행하라고 지시하는 것
  • "use strict"; 라는 문장을 스크립트 맨 위에 추가하면 전체 스크립트에 스트릭트 모드 설정
  • 함수 하나만 스트릭트 모드로 실행하려면 함수 본문 맨 앞에 추가

3.3 변수

  • var 연산자는 변수를 로컬 스코프(local scope)에서 정의한다는 점을 반드시 기억
  • 함수 안에서 var 연산자를 생략하면 변수를 전역으로 정의할 수 있다

3.4 데이터 타입

  • ECMAScript의 다섯가지 기본 데이터 타입(원시-primitive-타입) + 객체
    • Undefined
    • Null
    • Boolean(불리언)
    • 숫자
    • 문자열
    • 객체 : 이름-값 쌍의 순서 없는 목록
  • 변수의 데이터 타입을 알아내기 위해선 typeof 연산자를 사용
  • Undefined : var를 써서 변수를 정의했지만 초기화 하지 않았다면 해당 변수에 할당됨
    • 하지만 아예 선언되지 않은 변수도 typeof 로는 undefined가 나옴
    • 그러므로 변수를 선언할 때는 꼭 초기화를 해주는 습관을 기르도록 하는 것이 좋음
    • 그러면 선언된 변수인지 선언되지 않은 변수인지 확인 가능
  • Boolean : 가장 많이 쓰이는 데이터 타입 중 하나로 true와 false 두가지 리터럴 값만 가짐
    • 값을 불리언으로 변환할 때 Boolean()함수를 사용
    • 어떤 타입의 데이터에서도 호출할 수 있고 항상 불리언 값을 반환
    • 자동 변환이 되므로 제어문에서 사용하는 변수에 대해 정확하게 파악 필요
    • 만약 실수로 객체를 제어문에서 쓰게 되면 (항상 참이므로) 애플리케이션의 흐름이 완전히 달라짐
  • 숫자 타입 : IEEE-754형식을 통해 정수와 부동소수점 숫자를 나타냄
    • 기본적으로 10진법 사용
    • 부동소스점 숫자 사칙 연산에서는 부정확한 값이 나올 수 있음
    • ex)0.1 + 0.2 의 값이 0.3이 안나옴. 반올림 에러 때문.
    • 숫자형 값 중에는 NaN이란 특별한 값 존재 : 숫자를 반환할 것으로 의도한 조작이 실패했을 때 반환, 에러를 반환하는 것이 아님
    • isNaN() 함수를 이용해 해당 값이 숫자가 아닌 값인지 검사
    • 숫자가 아닌 값을 숫자로 바꾸는 함수 : Number(), parseInt(), parseFloat()
    • parseInt의 두번째 매개변수로 진법 표시 가능 ex) parseInt("0xAF", 16) 16진수 의미
    • 늘 뒤에 10을 매개변수로 넣는 습관(평소에 10진법 사용하므로)을 들이면 좋음
  • 문자열 타입 :16비트 유니코드 문자의 연속, "" 나 ''로 감싸서 표현
    • 문자열 변환은 toString() 메서드를 사용
    • toString() 메서드를 호출할 값이 null이거나 undefined일 가능성이 있다면 String() 사용
  • 객체 타입 : 데이터와 기능의 집합 var o = new Object();와 같은 형식으로 사용
    • ECMAScript의 Object 타입은 이 타입에서 파생하는 모든 객체의 원형
    • Object 타입의 인스턴스는 Object 타입의 프로퍼티와 메서드를 전부 상속
    • hasOwnProperty(propertyName) ,toString(), valueOf()등의 프로퍼티와 메서드를 가짐

3.5 연산자

데이터 조작에 쓸 수 있는 연산자

단항 연산자
  • 증감 연산자 : 피연산자 다음에 쓴 증감 연산자는 문장을 평가한 다음에 적용됨 ex)age++;

공통적인 특징

  • 연산자는 정수에만 쓸 수 있는 것이 아니라 문자열이나 불리언, 부동소수점 숫자, 객체에도 사용 가능
  • 유효한 숫자 형태의 문자열에 적용하면 문자열을 숫자로 바꾼 후 연산
  • 유요한 숫자 형태의 문자열이 아니라면 변수의 값에 NaN이 저장
  • 값이 false인 불리언에 적용하면 해당 값을 0 으로 바꾼 후 연산
  • 값이 true인 불리언에 적용하면 해당 값을 1로 바꾼 후 연산
  • 객체에 적용하면 해당 객체의 valueOf() 메서드를 먼저 호출한 후 나온 값으로 연산, 만약 결과가 NaN이라면 toString() 메서드를 호출한 후 다른 규칙을 다시 적용
비트 연산자
  • ECMAScript에서 모든 숫자는 IEEE-754 64비트 형식으로 저장
  • 하지만 64비트를 그대로 사용안하고 값을 32비트 정수로 변환하여 비트 연산자를 적용한 후 결과를 다시 64비트로 변환
  • 기본적으로 모든 정수는 부호 붙은 정수로 표현됨
불리언 연산자
  • 논리 AND와 논리 OR에서 두 피연산자에 따른 결과가 달라짐
  • 피연산자 2개에 따라 단축연산이 일어날 가능성 존재함
    • 예) 논리 AND 연산에서 첫 번째 피연산자가 false 인 경우 두 번째 피연산자는 검증 안하고 넘어감. 이 경우 두 번째 피연산자가 선언되지 않은 변수를 사용한다 해도 에러 발생 안하고 넘어갈 수 있음. 위험성 존재
    • 예) 논리 OR 연산에서 첫 번째 피연산자가 true 인 경우 두 번째 피연산자는 검증 안하고 넘어감.
  • 논리 OR 연산을 이용해 변수에 null 이나 undefined가 저장되지 않게 할 수 있음
    • var myObject = peferredObject || backupObject;
    • 논리 OR (||)을 이용하여 선호하는 값이 없는 경우에 백업해둔 값을 지정할 수 있음.
동일 연산자(==), 일치 연산자(===)
  • 동일과 비동일 연산자(==, !=)는 비교하기 전에 타입을 변경
  • 일치와 불일치 연산자(===, !==)는 타입 변환 없이 비교

기타 연산자에 대한 내용은 다른 언어들과 비슷함, 다만 피연산자가 객체인 경우는 특이한 경우 존재. 반드시 찾아보고 할 것.

3.6 문장

  • if문, do-while문, while문, for문 사용 가능, break, continue 가능, switch 가느
  • for-in 문
    • 객체의 프로퍼티를 나열하는데 사용
    • 프로퍼티에는 순서가 없으므로 어떤 순서로 반환할지 예측 불가능
    • 브라우저 호환성을 위해 for-in루프를 실행 전 객체를 가리키는 변수의 값이 null이나 undefined인지 확인하는 편이 좋음
  • with 문
    • 코드의 스코프를 특정 객체에 고정
    • 아래와 같이 자주 참조 되는 부분(location객체)을 줄여서 쓸 수 있음 javascript with(location){ var qs = search.substring(1); var hostName = hostname; var url = href; }

3.7 함수

  • ECMAScript의 함수는 꼭 값을 반환하지 않아도 됨
  • return 문 뒤에 반환값을 쓰지 않는 경우 undefined 값을 반환
  • 함수의 매개변수
    • 매개변수 숫자를 따지지 않으며 데이터 타입도 체크하지 않음
    • 함수는 arguments라는 객체를 갖는데 이 객체를 통해 매개변수 값이 접근 가능
    • arguments.length를 통해 매개변수가 몇 개 전달되었는지 알 수 있음
    • arguments.length를 통해 매개변수 개수에 따라 다른 동작을 수행하게 할 수 있음 - 오버로딩은 아니지만 흉내내기 가능
    • arguments[]에 값을 지정해도 이름 붙은 매개변수는 변하지 않는다
    • 오버로딩이 없다
