자바스크립트 제대로 배우기 - 4.변수와 스코프, 메모리

자바스크립트 제대로 배우기3

이 포스팅은 페이스북 자바스크립트 제대로 배우기 그룹의 공부 내용을 요약한 것입니다.

4장 변수와 스코프, 메모리

  • 자바스크립트 변수는 느슨한 타입을 취함 - 특정 시간에서 변수는 특정 값을 가리키는 이름일 뿐임

4.1 원시 값과 참조 값

  • ECMAScript의 변수는 원시 값과 참조 값 두가지 타입의 데이터를 저장가능
  • 원시 값 - 단순한 데이터로 Undefined,Null,불리언,숫자,문자열
  • 참조 값 - 여러 값으로 구성되는 객체
원시 값과 참조 값의 차이
  • 참조 값을 다룰 때는 언제든 프로퍼티와 메서드를 추가하거나 바꾸고 삭제 가능
  • 원시 값에는 프로퍼티가 없지만 추가하려 해도 에러가 생기지 않음 - 하지만 접근하려 하면 undefined 반환
  • 참조 값을 변수에서 다른 변수로 복사하면 힙에 저장된 객체를 가리키는 포인터가 복사됨
매개변수 전달
  • ECMAScript의 함수 매개변수는 모두 값으로 전달됨
  • 개발자들이 착각하는 점 : 함수 안에서 객체를 조작했을 때 외부에도 반영됨을 보고 매개변수가 참조 형태로 전달되었다고 오해, 하지만 함수 매개변수는 모두 '값으로' 전달됨
  • 함수 매개변수는 지역 변수와 다를 것이 없다고 생각하자
타입 판별
  • typeof 연산자는 원시 값에 대해서는 타입을 정확히 알 수 있음
  • 객체에 대해서는 ECMAScript의 instanceof 연산자를 사용함
  • instanceof 연산자는 변수가 주어진 참조 타입의 인스턴스일 때 true를 반환
  • 'result' = 'variable' instanceof 'constructor'형태로 사용

4.2 실행 컨텍스트와 스코프

  • 실행 컨텍스트(Execution Context)는 '컨텍스트'라고 부름
  • 변수나 함수의 실행 컨텍스트는 다른 데이터에 접근할 수 있는지, 어떻게 행동하는지를 규정
  • 각 실행 컨텍스트에는 변수 객체(variable object)가 연결되어 있음
  • 변수 객체에는 해당 컨텍스트에서 정의된 모든 변수와 함수가 존재
  • ECMAScript 프로그램의 실행
    • 함수를 호출 -> 독자적인 실행 컨택스트 생성
    • 코드 실행이 함수로 들어갈 때 -> 함수의 컨텍스트가 컨텍스트 스택에 쌓임
    • 함수 실행이 끝남 -> 해당 컨텍스트를 스택에서 꺼내고 컨트롤을 이전 컨텍스트에 반환
  • 스코프 체인(Scope Chain) : 컨텍스트에서 코드를 실행하면 변수 객체에 스코프 체인이 만들어짐. 실행 컨텍스트가 접근할 수 있는 모든 변수와 함수의 순서를 정의.
    • 스코프 체인의 앞쪽은 항상 코드가 실행되는 컨텍스트의 변수 객체
    • 함수라면 활성화 객체를 변수 객체로 사용
    • 다음 순서는 해당 컨텍스트를 포함하는 부모 컨텍스트
    • 전역 컨텍스트에 도달할 때까지 순서가 이어짐
    • 전역 컨텍스트의 변수 객체는 항상 스코프 체인의 마지막
  • 외부 컨텍스트는 내부 컨텍스트에 대해 전혀 알 수 없다, 반대로 내부 컨텍스트는 스코프 체인을 통해 외부 컨텍스트 전체에 접근 가능
스코프 체인 확장
  • try-catch 문의 catch 블록과 with문 은 모두 스코프 체인 앞에 변수 객체를 추가
자바스크립트에는 블록 레벨 스코프가 없다

다른 언어와 다른 개념

  • 블록 레벨 스코프를 지원하는 언어(C언어 등)에서는 for문의 초기화 부분에서 선언한 변수가 오직 for문의 컨텍스트 안에서만 존재
  • 그러나 자바스크립트에서는 for문에서 선언한 i변수가 루프 실행이 끝난 후에도 존재 javascript for(var i=0; i < 10; i++){ do someting(i); } alert(i); // 10
  • 함수 내 var 키워드를 생략하면 함수를 호출한 다음부터는 그 변수에 접근 가능(전역변수로 쓰임)
  • 컨텍스트 안에서 식별자를 참조하려면 검색이 필요
    • 식별자는 먼저 로컬 컨텍스트에서 검색되고 찾지 못하면 상위 컨텍스트로 스코프 체인을 따라 이동하며 검색
    • 전역 컨텍스트의 변수 객체에도 없는 경우 정의되지 않은 것으로 판단
  • 식별자가 로컬컨텍스트에 정의되어 있으면 부모 컨텍스트에 같은 이름의 식별자가 있다 해도 참조할 수 없음

4.3 가비지 콜렉션

  • 자바스크립트는 실행 환경에서 코드 실행 중에 메모리를 관리 - 가비지 콜렉션 언어
  • 자바스크립트는 필요한 메모리를 자동으로 할당하고 더 이상 사용하지 않는 메모리는 자동으로 회수
  • 더 이상 사용하지 않는 변수를 식별하는 기준은 브라우저마다 다름
  • 표시하고 지우기 또는 참조 카운팅 방법 을 사용
  • 표시하고 지우기 방법을 대부분의 브라우저에서 사용
  • 참조 카운팅 방법은 즉시 순환 참조 문제 발생 - 문제를 피하려면 사용 후 null할당을 이용해 연결 제거 필요
    • 연결 제거 후에는 다음 가비지 컬렉터가 실행될 때 삭제되며 메모리도 반환됨
  • 일반적으로는 개발자가 메모리 관리를 신경쓰지 않아도 됨
  • 하지만 웹브라우저에서 사용할 수 있는 메모리는 일반적인 데스크톱 앱의 가용 메모리에 비해 매우 작음
  • 수동으로 전역 변수 및 전역 객체의 프로퍼티 등을 참조 제거해주면 메모리 관리 가능 - 페이지 성능 증가 가능
  • 변수에서 참조를 제거하면 다음에 가비지 콜렉션 실행 시 해당 메모리가 회수됨
